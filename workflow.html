AI 工作流

把商家运营经验封装成简单流程

当前版本用前端规则模拟 AI 输出，重点展示 AI Builder 的产品编排思路。

信息采集门店、客群、活动和平台。
商家画像判断小店运营场景。
目标识别拉新、复购、推新品等。
平台策略匹配朋友圈、小红书、抖音。
内容生成文案、脚本、回复。
反馈优化复盘下一轮活动。

Prompt 编排思路

  • 角色设定
  • 输入结构化
  • 场景判断
  • 平台适配
  • 输出格式约束
  • 风险检查
  • 用户反馈迭代

示例 Prompt 模板

你是校园周边小商家的运营助手。请根据店铺类型、目标用户、活动目标、活动内容和发布平台，生成可直接复制使用的运营内容。输出需要包含朋友圈文案、小红书标题、短视频脚本、顾客回复话术和活动风险提醒。表达要符合小商家真实运营场景，不要夸大效果，不要承诺无法保证的结果。

未来真实系统架构

前端页面 → API 服务 → Prompt 编排层 → 大模型接口 → 内容安全检查 → 结果存储 → 用户反馈优化

当前静态 MVP 边界

  • 不接真实大模型。
  • 不存服务端数据。
  • 不自动发布到平台。
  • 不做真实效果归因。
  • 所有生成内容为前端规则模拟。

后续技术扩展

  • 接入大模型 API。
  • 增加用户账号。
  • 增加店铺画像数据库。
  • 增加 RAG 场景模板库。
  • 增加内容效果反馈。
  • 增加多商家管理。
